## 1 · The theme system (dark is home)

Two live themes + one stunt theme, driven by `html[data-theme]` and CSS
custom properties. **Default = dark.** Persisted in `localStorage mv:theme`.

Every THEMED page (see roster below) must:
1. Load `<script is:inline src="/theme.js"></script>` in `<head>` BEFORE its
   `<style>` blocks (sync, tiny — it sets `data-theme` pre-paint, decides
   the load-fill, updates the theme-color meta, and mounts the toggle).
2. Define this variable block at the top of its `<style is:inline>` and use
   ONLY vars for color (no literal `#FAFAF7` / `#101010` outside the block):

```css
:root {
  --paper: #FAFAF7; --ink: #101010;
  --soft: rgba(16,16,16,0.66);      /* secondary text */
  --faint: rgba(16,16,16,0.62);    /* labels, meta   */
  --hair: rgba(0,0,0,0.14);        /* hairlines      */
  --tint: rgba(16,16,16,0.04);     /* image wells    */
}
html[data-theme="dark"] {
  --paper: #101010; --ink: #FAFAF7;
  --soft: rgba(250,250,247,0.62);
  --faint: rgba(250,250,247,0.55);
  --hair: rgba(255,255,255,0.16);
  --tint: rgba(250,250,247,0.05);
}
html[data-theme="red"] {
  --paper: #E0301F; --ink: #0A0A0A;
  --soft: rgba(10,10,10,0.78);
  --faint: rgba(10,10,10,0.72);
  --hair: rgba(0,0,0,0.25);
  --tint: rgba(0,0,0,0.06);
}
html { background: var(--paper); }
body { color: var(--ink); }
::selection { background: var(--ink); color: var(--paper); }
```

3. Focus outlines, borders, image hairlines: `var(--ink)` / `var(--hair)`.
   Keep the `<meta name="theme-color">` tag (theme.js rewrites its content).

THEMED ROSTER: index, work, about, project/[slug], lab, labs/games,
labs/plugins, labs/roadmap, privacy, me, resume (new). NOT themed (their own
worlds — do not touch): the seven sims, alternates.html, /alternates/*
exhibits, the dot proto, /labs/satisfactory.

`/theme.js` API (already built — consume, don't reimplement):
- `window.MVTHEME.current()` → 'light' | 'dark' | 'red'
- `window.MVTHEME.toggle()` → liquid-fills to the other base theme
- `window.MVTHEME.liquidTo(theme)` → promise
- `window.MVTHEME.redMode()` → red fill, ~2s hold, fade back (the ticker's
  finale calls this)
- The toggle button mounts itself top-right on every themed page (fixed,
  44px target, half-moon dot, aria-label). Pages add NOTHING for it.
- Load ritual: on a manual reload (navigation type 'reload') or the first
  visit of a session, the page first paints in the INVERSE theme and the
  real theme pours in bottom-up (~650ms, wavy top edge, slight overshoot —
  liquid, not blob). In-site navigation and back/forward: no fill. PRM: no
  fill ever, instant states.

## 5 · The ticker (rebuilt from scratch — /mv/ticker.js, already built)

Minimal at rest: `0342 visits` — Space Mono 10px uppercase tracking, faint,
mounts inline in `#mv-ticker-slot`, no cards, no seams. Click = one event
from a pool (no immediate repeats), escalating to a finale:
confetti · letters space-out-and-snap-back · scramble-to-"hi" · "please
stop" · digit slot-spin · full flip. After the 10th click → `MVTHEME.
redMode()` (red pours up, ink text, ~2s at peak, fades home) and the click
count resets. Excluded browsers still see the number; nothing counts them.
Pages change NOTHING — the `#mv-ticker-slot` contract is the same.
